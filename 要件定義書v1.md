# 要件定義書

## 1. プロジェクト目的

本演習の目的は、複数種類の積荷をコンテナへ効率的かつ安全に配置するアルゴリズムを検討し、その設計結果を定量的指標に基づき評価することである。「設計」と「評価」を分離し、データ連携を行うことで、システム設計の基礎であるインターフェース定義と疎結合の概念を理解する。

## 2. システム構成

本システムは、JSON形式のファイルを介して連携する2つのモジュールで構成する。

1. **レイアウト設計モジュール**: 積荷データと制約を入力とし、バンニング配置をJSON出力する。
2. **評価モジュール**: 出力されたJSONを読み込み、物理制約のチェックとスコアリングを行う。

## 3. 物理・制約条件

### 3.1 外部環境（コンテナ・積荷）

- **コンテナ (40ft)**: 12000(L) × 2300(W) × 2400(H) (mm) / 最大積載量 24,000 kg
- **積荷（3種類）**:
  - 小：760 × 1130 × 550 (mm)
  - 中：1490 × 2260 × 900 (mm)
  - 大：2550 × 2280 × 2355 (mm)
  - 各積荷にはランダムな重量（1,000kg ~ 15,000kg）と目的地IDが付与される。
- **回転ルール**: 天地固定（Z軸回転禁止）。水平方向（X-Y平面）での90度回転のみ許可する。

### 3.2 配置・運用制約

- **物理制約**: 積荷の重複禁止、コンテナ壁面からのはみ出し禁止、接地制約（宙に浮かせない）。
- **目的地制約**: 1つのコンテナに混載できるのは、同一の目的地IDを持つ積荷のみとする（絶対制約）。
- **重量制限**: 各コンテナの積載合計重量は 24,000 kg を超えてはならない。
- **最低充填率**: 各コンテナの容積利用率は **50%以上** でなければならない（設計目標・評価対象）。

## 4. データ交換フォーマット（JSON仕様）

座標系は、コンテナの左・手前・下の角を原点 (0,0,0) とし、幅を x、奥行（長手）を y、高さを z とする。

```json
{
  "project_info": {
    "team_name": "Team_Alpha",
    "execution_time_ms": 1250
  },
  "containers": [
    {
      "container_id": 1,
      "destination_id": "DEST_A",
      "total_weight": 18500,
      "items": [
        {
          "item_id": "P001",
          "size_type": "medium",
          "dimensions": {"w": 1490, "l": 2260, "h": 900},
          "position": {"x": 0, "y": 0, "z": 0},
          "weight": 1200,
          "is_rotated": true
        }
      ]
    }
  ]
}
```

※ position は積荷の最小座標（原点に最も近い角）を指す。

## 5. 評価指標とスコアリング

評価チームは以下の項目に基づき、総合スコア S を算出する。

### 5.1 基本評価（減点・失格項目）

- **失格（Score 0）**: 重複配置、壁面はみ出し、接地違反、目的地混在、重量オーバーのいずれかがある場合。
- **充填率ペナルティ**: 充填率 50% 未満のコンテナ1本につき大幅減点。

### 5.2 重心バランス評価（長手方向：Y軸）

ケースサイズの関係上、コンテナ内は1列ではなく横2列でケースが搭載される。そのため現実には前後（Y軸）だけでなく左右（X軸）の重量差も存在するが、本演習では左右方向の重心バランスは考慮外とする。

コンテナの幾何学的中心 Yc = 6000 に対する合成重量重心 Yg の偏差を評価する。

- **満点範囲**: ズレが全長の 10%（±1200mm）以内。
- **許容限界**: ズレが全長の 25%（±3000mm）以内。これを超える場合は不安定荷役として減点または失格。

### 5.3 効率性評価

- **コンテナ総数**: 使用本数が少ないほど加点。
- **平均容積利用率**: 全コンテナの平均充填率が高いほど加点。

## 6. 開発ガイドライン

- **設計チーム（チーム1）**: 充填率だけでなく、積む順番や位置を調整して重心を中央に寄せるアルゴリズム（例：Heuristic Packing, Genetic Algorithm等）を模索せよ。
- **評価チーム（チーム2）**: 浮いている荷物や重なっている荷物を検出するロジックを厳密に構築せよ。また、計算時間を記録し、アルゴリズムの効率性も可視化せよ。
