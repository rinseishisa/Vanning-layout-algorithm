# バンニングアルゴリズム分析とアプローチ提案

## 1. 他チーム（ブランチ）のアプローチ分析

各ブランチのコードを分析した結果、以下の特徴と課題が見えました。

### 1-1. `taiga` のアプローチ (3D Point Generation + Greedy Heuristic)
- **手法**: 配置済みアイテムの角座標を「候補点」とし、各荷物が「どの候補点に、どの向きで置けるか」を全探索してスコア付けする貪欲法（Greedy法）。
- **特徴**:
  - 重量の大きい順・体積の大きい順にソートして配置。
  - Z軸（高さ）への積み上げに対応（下の荷物による支持判定を実装）。
  - Y軸重心（CoG）の制約を評価関数に組み込み、重心が中央から外れる位置にはペナルティを与えている。
- **課題**: 局所最適な配置を繰り返すため、後半に置く荷物の帳尻合わせが難しくなり、コンテナ終端付近で無駄なスペース（デッドスペース）が発生しやすい。また、順番に依存するため、偶然の並びに左右されやすい。

### 1-2. `sisa` のアプローチ (2D Free Rect / Guillotine Packing)
- **手法**: コンテナの床面積を「空き矩形（Free Rect）」として管理し、荷物を配置するたびに矩形を分割していく手法。
- **特徴**:
  - 体積の大きい順にソートし、Best-Fit Decreasing（余り面積が最も少なくなる矩形を選択）で配置。
  - Z軸=0（床置き）のみを前提としており、高さ方向への積み上げを行っていない。
  - コンテナサイズも20ft（L=5900）をベースに作られており、メインの要件（40ft）と乖離がある。
- **課題**: 3Dの空間を全く活かせていないため充填率が極端に低くなる。また、重心バランス（CoG）を考慮するロジックが存在しない。

## 2. Webリサーチ：現在主流となっているアルゴリズムと手法

3Dビンパッキング（Container Loading Problem: CLP）および重心（Center of Gravity: CoG）制約を扱う場合、世界的には以下のようなアプローチが主流です。

### 2-1. Extreme Point (EP) Heuristic
- 空間の角（Extreme Point）を管理し、そこに荷物を詰め込んでいく手法（taigaのアプローチの進化版）。単なる荷物の角だけでなく、壁面との交点なども管理し、空間を無駄なく使うことができます。

### 2-2. Block-Building (層・ブロック構築法)
- 荷物を一つずつ置くのではなく、同じサイズ・似たサイズの荷物をまとめて「ブロック（または層）」を作り、そのブロック単位でコンテナに配置していく手法。
- **メリット**: 空間利用率が非常に高く、積荷の安定性（崩れにくさ）が担保しやすい。

### 2-3. Two-Stage Algorithm (重心最適化の分離)
- **第1段階**: まず荷物のグループ（重い荷物・軽い荷物など）をコンテナのどこ（前・中・後）に配置するかというマクロな重心バランスの計画を立てる。
- **第2段階**: その計画に従って、ブロックごとの密なパッキングを行う。

### 2-4. Meta-heuristics (メタヒューリスティクス)
- 遺伝的アルゴリズム（GA）や焼きなまし法（SA）などの最適化アルゴリズム。
- 荷物を「投入する順番」や「回転のさせ方」の配列を遺伝子と見立て、何度もシミュレーションを繰り返して、充填率と重心バランスのスコアが最も高いものを探索する手法。

## 3. rui (私) 向け「一味異なる」アプローチの提案

既存の2チームが「上から順番に置いていく貪欲法」であるため、**「大局的な計画」や「探索」を持たせること** が、最も明確な差別化であり、最高の結果を出すための鍵になります。

- **アイデアA：メタヒューリスティクス（遺伝的アルゴリズム）の導入**
  - taigaのアプローチ（配置ルール自体は優秀）を評価関数として再利用し、「どの順番で荷物をコンテナに投入するか」を最適化する。
- **アイデアB：Block-Building ＋ Two-Stage 配置**
  - 先にブロックを仮想的に構築し、コンテナの両端（奥と手前）から交互に重い荷物をシーソーのように配置していく。
- **アイデアC：Wall-Building (壁構築法)**
  - コンテナの最奥から手前に向かって、壁を層のようにつくっていく手法。

## 4. 今後の開発方針（決定事項）

以上のリサーチ結果を踏まえ、ruiブランチでは以下の2つのアプローチを並行して進める方針とします。

1. **【メイン方針】アイデアA：遺伝的アルゴリズム（GA）による最適化**
   - `taiga` ブランチが実装している「基礎配置ロジック（候補点の列挙や、重なり・はみ出し・接地の判定）」自体は非常に優秀で堅牢です。
   - そこで、**taigaの基礎ロジックを「配置の評価エンジン」として流用**します。
   - その上で、「荷物をコンテナに投入する順序（および回転状態）」を遺伝子と見立ててGA（遺伝的アルゴリズム）を回し、重心バランスと充填率を極限まで高める「探索型ロジック」をメインアプローチとして開発します。

2. **【サブ方針】アイデアB：シーソー型ブロック配置（Two-Stage + Block-Building）**
   - アイデアA（探索型）の弱点である計算時間を補完し、論理的なパズル解決能力を高めるため、ルールベースのアプローチも並行して検証します。
   - サイズの揃ったブロックを事前に構築し、コンテナの手前と奥からバランスを取りながら積み込むロジックを構築します。
